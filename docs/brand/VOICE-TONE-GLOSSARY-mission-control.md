# Voice & Tone Glossary: Mission Control
**Rule 217 Deliverable**  
**Slug:** `mission-control`  
**Version:** 1.0  
**Created:** 2026-03-07  
**Core Principle:** Sound like a seasoned portfolio manager explaining to a trusted peer — not a company closing a deal.

---

## 1. Overview

**Product:** Mission Control — Fully automated institutional-grade trading system  
**Audience:** Sophisticated retail investors, 35–60, $250K–$2M investable, technically literate but not coders, time-constrained  
**Voice mission:** Write every line as if you're talking to a fellow serious investor at a Bloomberg terminal demo — not a prospect at a trade show.

**The Golden Rule (Rule 217):** Would you say this to your target user at their investing community meetup? If not, don't write it.

---

## 2. Voice & Tone Framework

### 2.1 Voice Attributes (Constant Across All Channels)

| Attribute | What It Means | What It Is NOT |
|---|---|---|
| **Precise** | Every claim has a number or a mechanism behind it. "147 risk gates" not "robust risk management." | Vague benefit language ("powerful," "innovative") |
| **Authoritative** | Speaks from evidence and track record, not promise. Doesn't over-explain or over-justify. | Defensive, over-qualified, hedge-everything |
| **Composed** | Calm under drawdown. Never alarmist. Doesn't escalate emotional language to sell. | Urgency tactics, fear-of-missing-out framing |
| **Transparent** | Surfaces the hard stuff (drawdown, risk, no guarantees) before the user has to ask. | Burying caveats in fine print |
| **Direct** | One claim per sentence. No weasel words ("may," "could potentially"). Owns the statement. | Passive voice, corporate hedging |

### 2.2 Tone Adjustments by Context

The voice stays constant. The tone shifts based on what the user is doing.

| Context | Tone | Key Words to Lead With |
|---|---|---|
| **User is learning** (first visit, reading strategy) | Patient, explanatory, peer-level | "Here's how it works…", "What this means in practice…" |
| **User is evaluating risk** (drawdown, black swans) | Honest, grounded, measured | "Yes, this can happen…", "The system is designed to reduce, not eliminate…" |
| **User is reading performance data** | Specific, evidence-led, unemotional | Numbers first; context second |
| **User is at CTA / decision point** | Confident, low-pressure | "If you're ready…", "Request access when it makes sense for you" |
| **User sees an error or system alert** | Clear, actionable, non-alarming | "Regime detected: UNFAVORABLE. New entries paused. No positions closed." |
| **User sees a good performance day** | Grounded, not giddy | "Solid session." not "Incredible results!! 🚀" |

### 2.3 What the Voice Is NOT

- ❌ **Startup hype:** "Game-changing," "disruptive," "revolutionary," "next-generation"
- ❌ **Toxic positivity:** "Trading is easy with the right system!" — No, it's hard; we manage that risk
- ❌ **Financial media breathlessness:** "MARKETS GOING CRAZY" style urgency
- ❌ **Generic SaaS:** "Powerful platform," "seamless integration," "robust solution"
- ❌ **Condescending simplification:** Over-explaining what beta or VIX mean to this audience
- ❌ **Liability-driven emptiness:** Stuffing every sentence with disclaimers

## 3. Language Glossary

### 3.1 Terms Sophisticated Investors Actually Use (✅ Use)

| Term | Context / Notes |
|---|---|
| **"regime"** | Market regime — SPY/VIX conditions that determine strategy mode |
| **"gate"** / **"risk gate"** | Pre-trade filter that must pass before an order goes through |
| **"NLV"** | Net liquidation value — standard IBKR/institutional term |
| **"drawdown"** | Max peak-to-trough decline — never "losses" in this context |
| **"premium income"** | Revenue from selling options — not "options profit" |
| **"covered call"** / **"CSP"** | Standard terms; no need to define for this audience |
| **"delta-neutral"** | Positioning language; use when describing exposure management |
| **"notional"** | Total market exposure value — preferred to "value of positions" |
| **"fill"** | Order execution — as in "every fill is logged" |
| **"roll"** | Options position management (roll to a later expiry) |
| **"run rate"** | Annualized projection based on current pace |
| **"walk-forward"** | Validation method; signals rigorous backtest to experienced investors |
| **"out-of-sample"** | Backtest period not used in parameter optimization |
| **"kill switch"** | Halt mechanism — keep this exact term; it's concrete and honest |
| **"morning brief"** | Daily summary — informal, implies you start your day with it |
| **"audit trail"** | Log of every decision — leads with transparency |

### 3.2 Terms to Avoid (❌ Avoid → ✅ Use Instead)

| ❌ Avoid | ✅ Use Instead | Why |
|---|---|---|
| "Guaranteed returns" | "Historical performance: 31% annualized" | Legally and ethically false |
| "Our platform" | "Mission Control" / "the system" | "Platform" is the most generic SaaS word |
| "Powerful" | State the specific power: "147 gates, 6 strategies" | Meaningless modifier |
| "Robust" | "Multi-layer" / describe the layers | Another empty adjective |
| "Seamless" | Never use | Always vague, always wrong |
| "Leverage the power of" | Delete entirely | Corporate filler |
| "Journey" | "Process," "how it works," or nothing | The investor isn't on a journey; they're managing capital |
| "Solution" | "System" / "Mission Control" | "Solution" implies a problem they haven't named |
| "Gain exposure to" | "Trade," "enter a position in" | Euphemistic |
| "Losses" (when describing drawdown) | "Drawdown" / "peak-to-trough decline" | More precise; less emotionally loaded in context |
| "Revolutionary" / "game-changing" | Cite the specific innovation | Hype language erodes trust with this audience |
| "Simple" (as main benefit) | "Zero manual trades" / "you review the brief" | "Simple" undersells; show the simplicity, don't claim it |
| "24/7 monitoring" | "Runs continuously during market hours" | Technically precise; avoids implying human eyes 24/7 |
| "Fully managed" | "Fully automated" | "Managed" implies human discretion; "automated" is honest |

### 3.3 Phrases to Use (✅)

| Phrase | When to Use |
|---|---|
| "Every decision is logged." | Any transparency claim |
| "The audit trail is always open." | Dashboard features, trust section |
| "No manual trades. Ever." | Automation benefit — absolute claim is a feature, not a bug |
| "~$8,600/month in premium income" | Performance section — always use tilde to signal approximation |
| "Six strategies. One pipeline." | Concise system summary |
| "The system doesn't sleep." | Emphasizes automation without anthropomorphizing too much |
| "Passes 147 risk gates before every entry" | Specific, concrete, credible |
| "Two independent classifiers run every morning." | Regime detection — specific and verifiable |
| "You review the morning brief." | User's role — positions them as overseer, not operator |
| "Solid session." / "Quiet day in the market." | Post-session communication — grounded, not excitable |

### 3.4 Language Patterns to Avoid

- ❌ **Emoji in any product or marketing copy** — this audience reads Bloomberg, not TikTok
- ❌ **Rhetorical questions that pressure** ("Why are you still trading manually?") — presumptuous
- ❌ **"We're excited to announce…"** — announce it; skip the excitement declaration
- ❌ **All-caps for emphasis** — NEVER. Use bold or sentence restructuring.
- ❌ **Stacking superlatives** — "the best, most comprehensive, fully-featured…" — pick one specific claim
- ❌ **"You deserve…"** — patronizing to this audience
- ❌ **"Finally, a solution that…"** — overused, implies the user has been suffering

## 4. Context-Specific Tone

### 4.1 When the User is Skeptical About Backtests

**Tone:** Transparent, pre-empt the objection, show methodology  
**Language:**
- ✅ "The 2-year backtest uses out-of-sample data — periods the optimizer never saw."
- ✅ "Walk-forward validation was used throughout."
- ✅ "Live account results are compared to backtest projections monthly."
- ❌ "Our backtests are more accurate than competitors."
- ❌ "We've proven the strategy works."

### 4.2 When Explaining a Bad Day / Drawdown

**Tone:** Measured, honest, contextual — never defensive  
**Language:**
- ✅ "Drawdown this week: -2.1%. S&P: -3.8% same period. System operated as designed."
- ✅ "Max drawdown historically: -18%. That's the honest number."
- ❌ "Despite challenging market conditions, our system performed admirably."
- ❌ "This was a temporary setback in our journey."

### 4.3 When Introducing Risk

**Tone:** Direct, first-person accountability, no burying  
**Language:**
- ✅ "This system manages risk — it doesn't eliminate it."
- ✅ "In a rapid crash, existing stop-losses hold. No system eliminates crash risk."
- ❌ "Past performance is not indicative of future results." (as the only risk statement — too boilerplate)
- ❌ Footnote-only risk disclosure

### 4.4 When at the CTA / Sales Moment

**Tone:** Low-pressure, confident, peer-to-peer offer  
**Language:**
- ✅ "Request access when it makes sense. The system is running either way."
- ✅ "See the live positions, the audit trail, and the morning brief. No commitment required."
- ❌ "Don't miss out — spots are limited!"
- ❌ "Join thousands of satisfied investors today!"

### 4.5 System Alerts / Error States (3-part format per Rule 214)

**Format:** What happened → Why → What to do  
**Language:**
- ✅ "Regime detected: UNFAVORABLE. New position entries paused. No existing positions affected."
- ✅ "Dashboard data stale (>5 minutes). The aggregator may not be running. Check system health in the admin panel."
- ❌ "Error: data unavailable." (no context, no action)
- ❌ "Something went wrong." (never use in a trading context)

## 5. Before / After Examples (Rule 217)

### Example 1: Hero Headline

❌ **Before:**
> "The Best Automated Trading Platform for Serious Investors"

✅ **After:**
> "Your money works 24/7. You review the morning brief."

**Why better:** Specific outcome (what the system does + what you do). No superlative ("best"). Shows the role split clearly.

---

### Example 2: Risk Management Benefit Card

❌ **Before:**
> "Advanced Risk Management
> Our powerful, multi-layered risk management system protects your portfolio from adverse market conditions."

✅ **After:**
> "Multi-Layer Risk Management
> 147 pre-trade checks, real-time stop-losses, portfolio-level drawdown limits, and a kill switch you control. Risk is never an afterthought — it's the first gate."

**Why better:** Specific count (147). Named mechanisms. Active voice. Ends with a positioning statement, not a feature list.

---

### Example 3: Comparison Table Cell

❌ **Before (generic):**
> "Yes" for "Risk management"

✅ **After:**
> "✓ 147 gates" (with hover tooltip: "Pre-trade regime check, drawdown gate, position sizing, sector concentration, correlation — checked before every entry")

**Why better:** Specific and verifiable vs. a check mark anyone can claim.

---

### Example 4: FAQ Answer

❌ **Before:**
> "Q: What happens if the market crashes?
> A: Our system has multiple layers of protection designed to safeguard your investments during extreme market events. Please be aware that past performance doesn't guarantee future results."

✅ **After:**
> "Q: What happens during a market crash or black swan event?
> A: Regime detection is designed to catch deteriorating conditions early. In a rapid crash, Layer 1 switches to STRONG_DOWNTREND or UNFAVORABLE, halting new entries. Existing positions are managed by stop-loss orders already placed at IBKR. No system eliminates crash risk entirely — the goal is to lose significantly less than an unmanaged portfolio."

**Why better:** Specific mechanism (Layer 1, STRONG_DOWNTREND). Specific action (stops already placed). Honest close (not a guarantee). No boilerplate disclaimer as a fig leaf.

---

### Example 5: CTA Button

❌ **Before:**
> "Learn More"

✅ **After:**
> "Request dashboard access →"

**Why better:** Verb-first. Describes exactly what happens. Arrow adds direction. No ambiguity.

---

### Example 6: Morning Brief Subject Line

❌ **Before:**
> "Your Daily Market Update - Mission Control"

✅ **After:**
> "Mission Control brief: 3 entries, 1 exit, regime CHOPPY — Mar 7"

**Why better:** Every piece of information is useful. Tells the investor the day's activity + current regime before they open it. No filler ("daily market update").

## 6. Quality Checklist

Use before publishing any copy:

### Voice Check
- [ ] Does this sound like a fellow serious investor, not a company?
- [ ] Would I say this at an investing roundtable without cringing?
- [ ] Is the tone appropriate for the context (learning / evaluating / deciding)?
- [ ] Is there at least one specific number, mechanism, or named term?

### Language Check
- [ ] Have I replaced all abstract adjectives (powerful, robust, seamless)?
- [ ] Is there any hype language (revolutionary, game-changing, disruptive)?
- [ ] Are error/alert messages using the 3-part format (what / why / how to fix)?
- [ ] Are CTAs verb-first with described outcomes?
- [ ] Does anything read like it came from a template?

### Authenticity Check
- [ ] Am I surfacing the drawdown/risk honestly, not burying it?
- [ ] Is this claim verifiable (number, mechanism, or attributable result)?
- [ ] Would this resonate with someone who has read "Margin of Safety" and uses thinkorswim?
- [ ] Is the tone peer-to-peer, not expert-to-novice or company-to-customer?
